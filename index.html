ThemeForest Product Development and Approval Process



Introduction:


This documentation provides a comprehensive guide to the process of developing, validating, and submitting HTML templates for approval on ThemeForest. The document outlines roles, responsibilities, tasks, timelines, and contingencies to ensure a seamless and efficient workflow.


The goal is to align product quality with ThemeForest's standards, achieve approval, and minimize rejection chances. This roadmap is intended for team members and stakeholders to understand the effort and time investment required for a successful submission.




Roles and Responsibilities:


  1. UI/UX Designer's Role


    • Objective: To craft a visually appealing and user-friendly design that adheres to modern standards and ThemeForest's expectations.
    • Key Deliverables:
      • Design files in tools like Figma, Adobe XD, or Sketch.
      • Responsive layouts for desktop, tablet, and mobile views.
      • Design guidelines and assets for development (e.g., fonts, colors, icons).
    • Notes: The designer ensures the quality of the visual and user experience, which significantly impacts the approval process. Any issues at this stage may lead to soft or hard rejection.

  2. Frontend Developer's Role


    • Objective: To convert the approved design into a fully functional HTML template ready for ThemeForest submission.
    • Key Deliverables:
      • Clean and valid HTML Markup, CSS Styling, and JavaScript and Jquery dom manipulation files.
      • Error handling
      • Documentation for end-users and reviewers.
      • Submission package adhering to ThemeForest guidelines.


Detailed Workflow


  1. UI/UX Design Phase (By Designer)


    • Description: In this phase, the designer creates a design that is not only aesthetically pleasing but also functional and in line with current industry trends.
    • Tasks:
      • Conduct research on trending designs in the ThemeForest marketplace.
      • Create wireframes or prototypes.
      • Iterate designs based on feedback.
    • Challenges:
      • Staying updated with ThemeForest's evolving design standards.
      • Balancing creativity with functionality.

    Estimated Times : 2–3 weeks.

    This time will be added to the UI/UX designer field.



  2. Development Phase (By Frontend Development)


    3. HTML, CSS, and JavaScript Development

    • Description:Transform the design into a fully coded template. This phase involves markup creation, styling, and adding interactivity.
    • Tasks:
      • Write semantic, well-structured HTML.
      • Implement CSS for layouts, animations, and styles(pre-processing) and Bootstrap Framwork.
      • Add functionality with JavaScript or jQuery, ensuring minimal library use for performance.
    • Tools: Visual Studio Code, Chrome DevTools, Git, and Gulp/Webpack for automation.

    Estimated Times : 3–4 weeks.


    Testing and Validation

    • Description:Ensure that the product works seamlessly across devices, browsers, and screen sizes.
    • Tasks:
      • Validate code using tools like W3C Validator for HTML, CSS Validator for styles, and ESLint for JavaScript.
      • Conduct cross-browser testing on Chrome, Safari, Firefox, Edge, and Opera.
      • Test responsiveness on various screen resolutions using tools like BrowserStack.

    Estimated Times : 5–7 days.

  3. Documentation Creation


    • Description:Provide clear, professional, and user-friendly documentation for the template.
    • Tasks:
      • Create a landing page that includes:
        • Features of the template.
        • Setup instructions.
        • File structure overview.
        • Contact/support information.
      • Prepare the following files:
        • description.html: A brief description of the template.
        • Thumbnail.png: A pixel preview image.
        • Theme-Preview.zip: live demo or preview link.
        • edit.txt: A changelog or update log.
    • Tools: Markdown editors, graphic design tool Photoshop or Canva and live preview generators.

    Estimated Times : 4–5 days.

  4. Submission and Review


    5. Submission


    • Description: Upload the template package on ThemeForest, adhering to their requirements.
    • Tasks:
      • Prepare and compress the submission folder.
      • Log in to the Envato Author Dashboard and upload the files.
      • Fill out metadata fields, such as the template name, description, tags, and demo URL.

    Estimated Times : 1 days.

    Review Process


    • Description: ThemeForest reviews the submitted template to check its design and technical quality.
    • Outcome:
      1. Approval: The product is published for sale.
      2. Soft Rejection: Minor issues are highlighted, which need to be fixed and resubmitted.
      3. Hard Rejection: Major flaws prevent the product from being approved.

    Estimated Times : 7–10 days.

    This time of the Review Process is for the Reviewers of Themeforest. This will not be added to the Developer's time.




    Key Factors for Approval


    • High-Quality Design: A professional, modern, and user-friendly UI.
    • Clean Code: Semantic, validated, and well-commented HTML, CSS, and JavaScript.
    • Comprehensive Documentation: Easy-to-follow user instructions and file descriptions.
    • Marketplace Research: Ensuring the product aligns with current trends and fills a niche.



    Common Rejection Reasons and Solutions


    Soft Rejection


    • Examples:
      • Poor code formatting.
      • Missing or incomplete documentation.
      • Minor responsiveness issues.
    • Solution: Review feedback, fix issues within 1–2 days, and resubmit.

    Hard Rejection


    • Examples:
      • Outdated or low-quality design.
      • Significant technical flaws.
    • Solution: Revise the entire product or start from scratch.


  5. Overall Timeline


    6. Phase Responsible Time Required
    UI/UX Design UI/UX Designer 2–3 weeks
    Frontend Development Frontend Developer 3–4 weeks
    Testing and Validation Frontend Developer 5–7 days
    Documentation Creation Frontend Developer 4–5 days
    Submission and Review Company Time 7-10 days

    Total Estimated Time: 5 - 6 weeks. (excluding review and ui ux design).

  6. Salary and Working Conditions


    • Expected Minimum Salary: 30,000 BDT per month.
    • Holidays: All government-declared public holidays will be observed as leave days.

  7. Conclusion


    8. This streamlined workflow guarantees an efficient process for developing and submitting templates to ThemeForest. By adhering to these guidelines, the chances of approval are significantly increased. For further assistance, feel free to reach out.
